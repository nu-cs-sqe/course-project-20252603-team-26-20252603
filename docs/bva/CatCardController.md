# BVA Analysis for `CatCardController`

## Method under test: `public Card play(Player currentPlayer, Player targetPlayer, int firstCardIndex, int secondCardIndex)`

## Purpose

This method implements the Cat Card action: when a player plays two matching Cat Cards, both Cat Cards are removed from the current player’s hand and added to the discard pile. Then the current player steals one random card from the target player.

`Player.canSubmitCard()` only checks whether the Cat Card is playable. This controller performs the actual game action.

## Notes

* `Player.canSubmitCard()` remains a playability check.
* `CatCardController.play(...)` performs the actual state-changing action.
* Random stealing should use an injected `Random` object so tests can control which card is stolen.
* When removing two played Cat Cards, the controller should remove the larger index first to avoid index-shifting bugs.
* Null values are not included as BVA cases for this controller because controller methods should not receive null during normal game flow. Null validation should stay at public domain boundaries where needed.
